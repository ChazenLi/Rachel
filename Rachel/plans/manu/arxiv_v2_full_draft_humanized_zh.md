# Rachel：一种面向多步逆合成的形式化化学推理框架

## 摘要

多步逆合成常被表述为路线生成问题，但实际难点往往不在于提出某一步看似合理的拆解，而在于系统能否在多轮展开中持续维护路线状态、比较局部候选，并在写入主路线前完成必要的化学检查。模板方法提供了清晰的局部动作语义，却不擅长跨步组织；近来的大语言模型（LLM）显示出更强的高层规划能力，却也容易把关键决策淹没在自由文本中。本文提出 Rachel，一种面向多步逆合成的路线展开框架。Rachel 将规划组织为 `state -> action -> validation -> commit` 循环，在由断键与官能团互变（functional group interconversion, FGI）构成的结构探索空间中进行局部试探、验证与提交，并保留失败尝试及其原因。这样的写法并不是把逆合成推向抽象证明，而是把原本隐含在多步规划中的判断步骤显式化：工具层负责提供化学事实与约束，LLM 主要负责在已接地候选之上组织局部策略。实验表明：在单步语料上，`bond + FGI` 相比 `bond-only` 明显扩大了可达且化学上可接受的前体空间；在固定候选空间中，更丰富的结构化上下文主要改善了 LLM 对已有候选的选择，而不是促使其脱离工具约束自由生成前体；在 PaRoutes `n1` 的冻结设置下，Rachel 为 `51/51` 个目标生成了完整 route tree，其中 `42/51` 条路线在当前在线审计口径下达到闭合。相应地，本文也区分 `reference-route similarity` 与 `practical closure`：前者衡量与基准路线的接近程度，后者衡量路线是否真正推进到当前口径下可接受的终点。

## 1. 引言

逆合成分析关注如何把一个目标分子逐步回溯为更简单、也更容易获得的前体。过去几十年里，反应模板、反应数据库、启发式搜索和数据驱动模型共同推动了计算机辅助逆合成的发展 [1-6]。但强单步扩展器并不会自动带来可靠的多步规划。一个多步系统除了决定“下一步在哪里断”，还必须持续维护中间状态，比较局部候选，在断键与转化之间分配搜索预算，并在每一步写入主路线前完成基本化学检查。真正困难的地方，往往不在于提出一个听单步合理的拆解，而在于让整条路线在约束下稳定推进。

现有方法大多只覆盖了这一问题的一部分。模板与搜索方法保留了清晰的动作语义和较好的可控性 [2-4]，但在覆盖范围和长程策略表达上仍有限制。模板自由或神经方法显著扩展了候选空间 [5-8]，却更容易把中间状态和失败来源封装在模型内部。近来的 LLM 逆合成工作进一步展示了高层规划潜力 [9,10]，例如跨步比较局部方案、协调不同证据或处理软约束；但一旦关键决策主要停留在自由文本层，系统就很难清楚地区分：哪些动作只是被提议，哪些动作通过了检查，哪些尝试已经被否决。对多步问题而言，这一差别不是表述细节，而是决定路线能否被分析、复核和持续改进的基础。

本文的基本判断是，多步逆合成更适合被写成一种带状态、带验证、带提交记录的规划过程，而不是若干松散文本的串联。更具体地说，我们希望把多步逆合成从“生成一条答案路线”提升为“在化学约束下逐步构造、验证并提交路线的推理过程”。为此，真正参与规划的要素被显式放入系统内部：当前分子、待处理队列、路线树与决策记忆共同定义状态；断键、FGI 与模型提议的前体构成动作；原子守恒、正向验证、结构一致性和官能团兼容性共同界定可接受性；模板来源、局部决策记录、检查结果与失败痕迹则持续保留在路线中。这里所谓“形式化”，并不是把合成规划还原为完备逻辑，而是让每一步前体从何而来、为何被接受、又在何处失败，都能在后续分析中被重新核对。

Rachel 将这一判断落实为一个连续的 `state -> action -> validation -> commit` 循环。系统不要求模型一次性写出整条路线，而是通过多轮局部决策逐步推进；它也不依赖单一扩展模式，而是在断键与 FGI 组成的双重结构探索空间中行动。候选步骤先在局部工作区中试探，再决定是否写入主路线树；未被接受的尝试不会被抹去，而会作为失败记录留在决策历史里。这样的组织方式也对应一种更清楚的职责划分：工具层负责给出可检查的化学事实与候选动作，LLM 的职责则被收紧为在这些已接地候选之间比较、排序和组织局部决策，而不是脱离约束独立给出答案。

这种组织方式也改变了我们对“成功”的理解。精确回放 benchmark 中的参考路线仍然重要，但它并不是唯一目标。一个系统完全可能没有复现 reference route，却仍在固定预算和验证规则下找到一条能够通向可接受终止原料的路线。基于这一点，本文始终区分 `reference-route similarity` 与 `practical closure`：前者描述系统与基准路线有多接近，后者描述系统是否真正把目标推进到了当前口径下可接受的终点。后文的实验和结果，也将围绕这一区分来组织。

本文的贡献可以概括为四点。第一，我们把多步逆合成表述为一个由状态、动作、验证与提交组成的路线展开问题，使局部决策及其失败记录进入同一规划过程。第二，我们实现了 Rachel 框架，用 `bond` 与 `FGI` 两类结构探索算子、局部试探以及验证后提交的执行机制来组织多步路线生成。第三，我们将实验拆分为三个互补层面：局部动作空间覆盖、固定候选空间中的选择质量，以及冻结设置下的端到端闭合与未闭合边界。第四，基于当前已完成的单步与 `n1` 结果，我们得到三个有边界的经验判断：`bond` 与 `FGI` 更适合作为互补算子共同使用；LLM 在本系统中的主要价值是对已接地化学候选进行选择和组织；多步评测中，实用闭合与参考路线重放应分开报告。

## 2. 相关工作

### 2.1 基于模板与搜索的逆合成方法

基于模板与搜索的逆合成方法构成了现代 CASP 的主干。早期的 LHASA 将断键启发、官能团互换和保护基逻辑编码为可执行规则 [1]。Segler 等随后把模板扩展、神经排序和 Monte Carlo tree search 结合起来，说明模板库不仅能做静态检索，也能在学习到的优先级引导下支持完整多步规划 [2]。AiZynthFinder 将这一范式做成开放且可复现的平台，而 Retro* 则进一步把路线搜索写成神经引导的 A* / best-first 过程，用分子合成代价控制展开顺序 [3,4]。这些工作分别回答了模板优先级、搜索效率和路线排名等关键问题，也奠定了今天多步规划的基本框架。

但在这一路线里，被重点建模的通常仍是“扩展什么”和“先搜哪里”。至于某一步为什么被正式写入路线、某个前体为什么被拒绝，以及失败到底发生在模板覆盖、搜索预算还是终止物接地，往往没有作为路线本体的一部分被系统保留下来。Rachel 与这一传统的连续性，在于它同样把多步逆合成理解为搜索问题；它的差别在于，把前体尝试、化学检查和拒绝原因也纳入了路线展开本身。

### 2.2 模板自由、神经网络与 LLM 逆合成

另一条主线聚焦单步模型本身。Molecular Transformer 表明，基于反应 SMILES 的序列模型可以在不显式匹配模板的情况下学习反应变换 [5]；RetroXpert 将单步逆合成拆成反应中心识别与 synthon completion，更接近化学家先找断点、再补全前体的工作流 [6]；GraphRetro、LocalRetro 等图模型则把重点放在局部图编辑、反应中心和邻域反应性上，以更直接地刻画键级变化与官能团环境 [7,8]。这些方法显著提升了单步覆盖和排序质量，也说明高质量前体并不完全依赖手工模板库。

近来的另一条路线则是把语言模型引入化学决策。ChemCrow 展示了 LLM 与外部化学工具协作的通用框架，T-Rex 等工作进一步尝试让文本描述参与反应中心排序和逆合成决策 [9,10]。这类系统的吸引力在于，它们能够把分子描述、外部工具和跨步比较放进同一过程；但如果最后被接受的前体主要停留在自由文本层，那么前体是“被提议”还是“被采纳”、失败发生在哪一步、哪类检查真正通过，往往会重新混在一起。Rachel 与这一方向的区别不在于排斥 LLM，而在于把 LLM 放回路线设计和化学决策的位置，把前体提交与化学核查留给工具层和路线状态。

### 2.3 超越路线重建：多步规划的评测

PaRoutes 为多步逆合成提供了统一的 benchmark route set、stock setting 和 solved-rate 比较口径，使不同搜索器和单步模型能够在同一约束下比较 [11]。在此基础上，研究开始逐步超出 solve / not-solve 的二分视角：有工作从路线相似性、树结构或反应序列衡量与参考路径的接近程度，也有工作用 learned strategy、route scoring 或 Retro-BLEU 一类指标评价路线的可取性和整体质量 [11,12]。这一变化并不意外，因为对同一个目标分子而言，通常存在不止一条化学上可接受的路线。

Rachel 与这一趋势一致，但关注点更具体。本文不仅关心最终路线与基准路线有多接近，也关心前体是如何被挑出、在哪一步被接受，以及未闭合主要停在什么位置。因此，我们区分两类互不等价的结果：一类是与参考路线在步骤、边或拓扑上的接近程度，另一类是系统是否通过经检查的步骤把目标推进到了可接受终点。这个区分不是为了替较低的 replay 成绩辩护，而是为了更准确地界定多步规划系统究竟完成了什么。

### 2.4 Rachel 的位置

因此，Rachel 位于三类思路的交汇处：它继承了 Segler、AiZynthFinder 和 Retro* 一类方法对搜索与预算分配的重视，吸收了 Molecular Transformer、RetroXpert、GraphRetro 和 LocalRetro 一类单步模型对局部结构变化的刻画，也借鉴了 ChemCrow 以及近期 LLM planner 在跨步比较与工具协调上的灵活性。Rachel 的重点不是再造一个单步扩展器，而是把前体尝试、化学检查、终止接受和失败记录放进同一条路线中。与其说 Rachel 是另一个路线生成器，不如说它是在重新规定：一条多步路线里，哪些信息必须被保留下来，哪些判断应由工具承担，哪些判断又更适合交给模型来组织。

## 3. 形式化问题定义

Rachel 的出发点并不复杂。多步逆合成之所以难，并不是因为缺少一个更强的下一步预测器，而是因为规划过程本身需要更清楚的组织。基于这一点，我们不把逆合成视为若干文本建议的串联，而把它写成一个在分子与路线树上运行、并在每一步都受原子守恒、forward check 和官能团兼容性约束的状态化过程。换句话说，这里的形式化首先作用于规划对象本身：被持续更新的不是一个“答案”，而是一条包含待展开分子、局部候选、已提交步骤和失败记录的路线状态。这样处理的目的很务实：让前体选择、提交和回退都能够被明确记录，同时保留处理复杂目标所需的灵活性。

记时间步 \(t\) 的规划状态为

$$S_t = (Q_t, T_t, A_t, C_t),$$
其中 $Q_t$ 表示待处理分子队列，$$T_t$$ 表示当前逆合成路线树，$$A_t$$ 表示近期决策与失败记录，$$C_t$$表示当前分子的局部决策上下文。这样的状态写法包含两层含义。其一，多步逆合成本质上是状态依赖的：下一步动作不仅取决于当前分子结构，也取决于已经提交的步骤、当前分支布局、剩余预算以及累积的失败历史。其二，凡是与路线推进有关的关键信息，都应进入状态本身，而不是散落在隐含的对话记忆或松散的 prompt 历史中。

在这一状态空间上，Rachel 定义了一组具有明确化学语义的动作 $$\mathcal{A}$$。主要动作类型包括断键、官能团互变（FGI）以及模型提议的前体生成。断键动作对应于在规则或模板支持下直接进行骨架拆解；FGI 动作在不切断主骨架的情况下改写局部化学表征，从而改变后续哪些断裂变得可达；模型提议的前体动作则允许策略层在模板直接覆盖之外提出候选前体，但这些候选仍需经过后续检查。除此之外，Rachel 还包含 `accept-terminal`、`skip` 等控制类动作，用来明确标记终止与放弃。关键之处在于：动作不是自由文本，它们都有清楚的角色、输入、输出以及对状态的影响。

Rachel 同时将局部探索与全局提交明确区分开来。给定状态 \(S_t\) 与候选动作 \(a \in \mathcal{A}\)，系统首先在局部工作区中执行试探：

$$\mathrm{try}(S_t, a) \rightarrow r_t,$$

其中$r_t$记录局部结果，包括候选前体、结构分析、验证输出、原子平衡信息，以及在适用时的失败标注。只有当局部结果被检查并接受之后，系统才进行全局更新：

$$\mathrm{commit}(S_t, a) \rightarrow S_{t+1},$$

即将该步骤写入路线树，并同步更新队列、失败记录与局部决策上下文。`try/commit` 的分离十分关键，因为它允许系统在不污染主路线的前提下比较多个局部候选，也让回退成为路线中的可记录步骤，而不是 prompt 重写带来的隐式副作用。

在 Rachel 中，验证被放在规划循环的核心位置，而不是作为事后打分。一个候选步骤不会因为“看起来合理”或者“像某种常见反应”就被接受，它必须通过原子守恒、正向验证、骨架或拓扑一致性、官能团兼容性等检查。换句话说，每一次局部动作不仅要回答“提出了什么前体”，还必须回答“这组前体为什么能够成立”。这样一来，多步逆合成就不再是缺乏证据的建议堆积，而成为附带检查结果的路线构造。

这种形式化也改变了我们对“成功”的定义。Rachel 并不把目标压缩为与单一参考路线的重合度最大化，而是明确区分两类目标。第一类是 `reference-route similarity`，即预测路线与基准路线在步骤结构、拓扑或关键边上的相似性；第二类是 `practical closure`，即系统是否能够在固定预算下找到一条化学上可接受、通过验证且通向可接受终止原料的路线。前者对应的是对已有标注路径的贴近程度，后者更接近规划系统在真实使用中应当提供的能力。

因此，这里的形式化首先是操作性的，而不是逻辑意义上的。它并不试图把合成规划改写成抽象证明，而是试图把原本隐含在多步规划中的判断步骤显式化：哪些候选来自工具层的化学事实，哪些选择属于策略层，哪些步骤通过了验证，哪些失败停在了终止物接地。这样一种路线表示，不仅让局部动作、验证与提交进入同一过程，也使后续的评测能够区分“是否沿参考路线前进”和“是否真正把目标推进到可接受终点”。

## 4. Rachel 框架

Rachel 将上述问题表述落实为一个分层规划系统。对本文而言，关键不在于铺开所有实现细节，而在于说明 Rachel 如何把状态、化学动作生成、验证和策略决策放在同一条路线推进链条中。它并不是一个“LLM 加工具”的单体流水线，而是一个让事实、可接受性和策略保持分离、因而可以逐步核对每一步前体与提交条件的规划系统。

### 4.1 规划循环与状态

从整体上看，Rachel 从目标分子出发，反复执行上下文构造、动作提议、局部试探、验证与提交。它不要求模型用一条长 prompt 一次性写出整条路线，而是通过连续的局部决策逐步推进搜索，使每一步都有稳定语义，每一条路线也都能被保留下来并重新检查。

这一状态包含目标分子、待处理队列、当前路线树、近期决策记录以及局部工作区内容。这里的关键不只是接口形式，而是把多步逆合成真正做成一个可恢复的搜索过程：系统可以重新检查早先选择、比较局部备选，也可以在失败或预算耗尽后继续规划，而不是把整件事压缩成一次性的文本交换。

其中有两部分信息尤其关键。路线树记录已经被正式接受的内容，包括分子、反应、父子关系、动作类型，以及模板来源和验证摘要等证据；另一部分则记录系统如何走到当前阶段，例如近期决策、失败试探和算子偏好。两者结合起来，路线构造就不再只是一个最终答案，而成为一个带记忆、可追溯的过程。

### 4.2 双重动作空间与验证门控执行

Rachel 的化学核心始于一个直接的设计选择：LLM 不直接面对一个无约束的分子字符串，也不被要求像回答自由问答那样即兴给出化学方案。相反，当前分子会先被编译为结构化决策上下文，其中包含分子描述符、官能团、候选断键、FGI 机会、复杂度信号以及可行性提示。这个决策面建立在两类互补算子之上：断键主要回答“骨架可以在哪里被拆开”，FGI 主要回答“某个有价值的拆解是否需要先经过一次局部化学改写才会变得可达”。

这层双重性对本文的论点是关键的。Rachel 并不是先假定一个唯一扩展模式，再在它失效时打补丁。它确实采用了一个轻量级的断键优先先验来控制预算，但 FGI 仍是一等动作类型，而不是后备修补。对于那些切口看起来有希望、却暂时缺乏明确模板前体实现的情形，诸如 `smart_cap` 这样的辅助启发式可以提供 capped precursor 候选，而不会把系统推向无约束的前体发明。

局部探索则通过 `try/commit` 的两步处理完成。来自断键、FGI 或模型提议的候选前体，都会先在局部工作区中被执行和检查，系统在其中记录候选产物、复杂度变化、原子平衡分析、正向验证结果以及失败原因。只有通过相关检查的局部结果，才允许进入主路线树。这样的验证后写入同时实现了三件事：它让局部备选可以被并排比较，防止薄弱试探悄悄污染主路线，也把被拒绝的动作转化为可复用的失败记录，而不是 prompt 历史里的噪声。

### 4.3 LLM 作为策略层

在这套流程内部，LLM 的角色是刻意有边界的。它不负责界定化学合法性，也没有绕过验证的权力；这些职责始终留在化学算子和验证器手里。模型真正提供的是灵活的策略判断：比较局部方案、决定应优先断键还是转化、判断何时接受终止节点，以及在局部试探失败后调整后续选择。

因此，Rachel 将 LLM 放在整理后化学信息之上的策略层，而不是把它当作化学预言机。它的输入是从当前状态编译出的紧凑或扩展决策上下文，输出则是动作选择、探索决策或待验证的前体建议。实践中，模型最稳定的价值体现在：对已接地选项进行排序，并围绕这些选项组织局部决策序列。分工因此是清楚的：工具提供事实，验证器界定可接受性，LLM 则在合法动作空间内组织搜索。

从这个角度看，Rachel 既不是“模板枚举器外接一个 LLM”，也不是“带工具接口的化学聊天机器人”。它是一个以化学结构组织路线展开、并把模型判断放进明确搜索步骤中的系统。也正因为如此，高层规划、路线检查和失败分析才能在同一框架里被统一讨论。

## 5. 实验设置

实验设计围绕三个彼此衔接的问题展开：这套局部化学动作能否稳定覆盖真实拆解；在候选空间固定时，模型的选择会不会因上下文而改变；这些局部决策累积到多步层面后，最终得到的是怎样的路线及怎样的未闭合边界。相应地，正文依次报告单步覆盖、`compact/full` 对比以及 PaRoutes `n1` 上的端到端结果。这样的安排，是为了把化学基底、策略选择和路线闭合分开讨论，而不把它们压成单一总分。

### 5.1 实验概览

本文的实验分为三层。第一层检验由 `bond` 与 `FGI` 组成的局部化学基底，关注真实前体是否落在候选集合内，或至少是否存在能够通过 forward check 的可行前体。第二层在相同候选空间下比较 `compact` 与 `full`，考察更完整的局部证据是否改善前体选择。第三层在 PaRoutes v2 `n1` 的 `51` 个目标上观察多步路线的展开、闭合、与参考路线的偏离，以及未闭合主要发生在哪一层。

这样的分层安排，使三个层面的结论尽量彼此独立：单步部分用于确认 Rachel 是否拥有值得被规划器反复调用的局部动作；`compact/full` 对比用于区分“候选是否存在”和“候选是否被选中”；端到端部分则用于观察这些局部选择累积之后，路线最终停在中间体、终止物，还是仅仅偏离了 benchmark 记录的参考路径。

### 5.2 数据集与对比设置

单步分析使用两类语料：USPTO50K 全量单步反应，以及由 PaRoutes `n1/n5` 完整路线拆分得到的单步目标。这里的单步结果只用于判断局部化学动作是否足以支撑后续搜索，不作为多步规划能力的替代证据。算子层面的核心对比为 `bond-only` 与 `bond + FGI`，借此区分纯骨架断裂与“骨架断裂 + 局部官能团改写”在可达前体空间上的差别。

第 6.2 节的 blind structured-decision 实验基于 USPTO 中抽取的 `500` 个目标。两种设置共享完全相同的化学候选集合，变化的只有提供给 LLM 的局部证据。`compact` 仅保留压缩后的决策面，`full` 则保留更完整的 bond/FGI 选项和局部结构信息。因此，该实验衡量的是模型在既有候选中的辨别能力，而不是生成新前体的能力。

端到端评估聚焦 PaRoutes v2 `n1` 的 `51` 个固定目标。正文报告 Rachel 在这一目标集上的路线结果，并与 PaRoutes、RootAligned、RootAligned MCTS 和 LocalRetro 的公开 benchmark 结果对照。`n5` 暂不纳入主文，因为本文更关心在较小而受控的目标集上看清路线如何闭合，以及未闭合首先停在何处。

### 5.3 评测指标

单步部分主要报告 exact inclusion、`chemical_feasible` 和 near-reference valid。我们定义 `chemical_feasible = exact_correct + chemical_correct`，其中 `chemical_correct` 指候选前体经 forward check 可回生成目标产物；near-reference valid 则表示候选与记录参考答案保持较高相似。前者衡量这一步在化学上是否成立，后者反映它与数据集中已记录前体的接近程度。

`compact/full` 对比主要报告 reference replay exact、mean/median precursor-set Tanimoto、相似度分桶、solved rate，以及 `bond / FGI / custom` 的动作分布。这组指标共同描述模型在固定候选空间中的选择质量。化学上，更高的 Tanimoto 和更低的低相似尾部意味着模型更少落到表面合理但局部环境不符的拆解，而更常选中与离去基、杂原子连接位点和取代模式相一致的前体。

多步部分主要报告以下几类指标：跨方法 solved rate；Rachel 自身的完整路线树生成率、路线闭合率和终止分子通过率；与参考路线在树、反应集合和终止分子集合上的接近程度；共同可解目标上的路线深度、终止物数量和复杂度压缩；以及未闭合样本中缺失终止物的来源分布。这些指标分别对应路线是否真正走到终点、终点是否已经压向更简单的可采购前体，以及它与 benchmark 路线的差异主要落在步骤组织还是终止物层面。

### 5.4 固定设置与统计说明

除非另有说明，所有实验均固定目标池、模型版本、解码参数、搜索预算、终止条件和输出结构。多步部分对每个 `n1` 目标保留一条序列化运行轨迹；blind LLM 对比则将题目与答案严格分离，并确保 `compact` 与 `full` 在同一候选集合上作答。这样可以尽量把观察到的差异限制在候选覆盖、上下文呈现或路线闭合本身，而不是来自运行条件的漂移。

统计处理保持克制。正文以描述性结果为主：比例型指标用 Wilson 区间概括，Tanimoto 等连续量用 bootstrap 汇总；对于 `compact` 与 `full` 这类配对实验，更关注逐题变化方向和分布偏移，而不把重点放在单一显著性检验上。以本文当前的证据规模，这样的统计口径已经足以支撑主文结论。

## 6. 结果

结果部分沿着与系统设计相同的顺序展开。我们先看这套化学基底是否确实提供了可搜索的局部动作，再看在候选空间固定时更丰富的上下文究竟改变了什么，最后看这些局部决策在多步层面能否累积为闭合路线，以及未闭合主要停在何处。这样的顺序对应的不是叙事包装，而是证据之间的依赖关系：若局部候选本身不成立，后续的模型选择和端到端路线就无从讨论。

### 6.1 形式化基底在大规模单步语料上提供稳定覆盖

对于 Rachel 而言，最先需要确认的不是多步搜索能走多远，而是底层局部化学动作是否足够可靠：当真实前体不能被精确回放时，它是否仍能把化学上站得住的局部拆解放进候选空间。如果这个前提不成立，多步规划就缺乏值得搜索的对象。为此，我们同时考察 USPTO50K 全量单步数据，以及由 PaRoutes `n1/n5` 完整路线拆分得到的单步语料。这样做不是用单步结果替代多步评测，而是先回答 Rachel 所依赖的化学底盘本身是否值得被规划器反复调用。

在本节中，我们把能够通过 forward check 回生成目标产物的候选前体视为主要阳性信号，即 `chemical_feasible = exact_correct + chemical_correct`。作为对照，我们同时保留与记录参考答案的接近度：在 PaRoutes 上对应 Morgan Tanimoto `>= 0.9`，在 USPTO 上对应 legacy `Exact + NearMiss`。前者更接近“这一步在化学上是否站得住”，后者回答“它与数据集中记下的那组前体有多接近”；两者相关，但不必重合。

Fig. 1A 最直接的发现是，虽然 exact inclusion 在三个语料上都只有约 `19%`，`chemical_feasible` 却稳定落在 `57.1%` 到 `57.7%` 之间，而 near-reference valid 与它的差距始终只有 `1` 到 `4` 个百分点。这个组合很重要。它说明 Rachel 的单步基底并不擅长机械回放唯一答案，却能相当稳定地把化学上可接受的局部动作纳入候选空间；同时，这些候选又没有明显偏离数据集中记录的已知化学。对多步规划来说，这比追求单一前体集合的严格重现更关键，因为规划器真正需要的是一组可继续展开的局部动作，而不只是一个唯一正确标签。

这种稳定性也提示，这套基底捕捉到的并不是某个特定数据源的偶然模式。PaRoutes 上，near-reference valid 比 `chemical_feasible` 低 `1.27` 个百分点；USPTO 上则高 `4.35` 个百分点；合并后两者仅相差 `1.02` 个百分点。换句话说，不同来源的单步数据虽然记录方式和参考答案口径并不完全一致，但 Rachel 给出的局部候选在“化学上成立”和“与已记录前体接近”这两层上都保持了相近趋势。

如果进一步追问这约 `57%` 的覆盖从何而来，USPTO50K 上的算子对比给出了更有化学意味的解释。Fig. 1B 表明，`bond` 与 `FGI` 解决的并不是同一个问题：前者主要提供显式骨架断裂，后者则补上那些只有在局部官能团先被改写后才变得可达的拆解。在这一口径下，`bond + FGI` 将 exact inclusion 从 `8.92%` 提高到 `19.54%`，把平均 precursor-set Tanimoto 从 `0.800` 提高到 `0.878`，并在 `13,793 / 48,223 = 28.6%` 的样本上相对 `bond-only` 带来改善。

这意味着，Rachel 的化学底盘之所以成立，并不是因为某一个核心算子单独足够强，而是因为骨架拆解与局部官能团改写在很多目标上本就需要配合使用。对后文的 `compact/full` 比较而言，这一点尤其关键：blind structured-decision 所依赖的不是一个稀薄的模板背景，而是一套已经具有实质覆盖能力的已接地候选空间。事实上，在用于 blind structured-decision 的 500 条子集上，template exact rate 已达 `47.8%`，template partial rate 为 `78.0%`，平均 template Tanimoto 为 `0.925`。换句话说，6.1 支持的首先不是“单步命中率还不错”，而是 Rachel 确实拥有一套值得被持续调用的局部化学动作空间；后文真正检验的，是模型如何在这套空间之上组织选择，而不是系统是否拥有这些候选。

### 6.2 更丰富的上下文改善选择，而不是扩张化学空间

在确认化学基底本身具有稳定覆盖之后，问题就可以被问得更精确：当候选化学动作保持不变时，LLM 究竟增加了什么。`compact` 与 `full` 的比较之所以重要，正是因为它尽量把“化学候选的生成”与“候选的组织和选择”分开来考察。两种设置共享同一工具层动作空间；变化的只是局部证据被呈现给模型的方式，以及模型在决策时能够看到的结构化上下文密度。

这种差异先体现在输入侧。Fig. 2A 压缩了 `compact` 与 `full` 的提示词暴露差异：`full` 的平均 prompt 长度是 `compact` 的 `2.32x`，平均行数达到 `3.02x`，而可见 bond 与 FGI 选项数也分别扩展到 `1.33x` 和 `1.87x`。这里重要的并不是“prompt 更长”这一表面事实，而是模型在做同一题时看到的结构化证据确实更完整了。

输出侧的变化更有信息量。Fig. 2B 表明，`full` 明显改善了分解质量：mean Tanimoto 从 `0.624` 提高到 `0.677`，median Tanimoto 从 `0.553` 提高到 `0.629`，低相似尾部 `<0.5` 降低了 `8.0` 个百分点，而 `=1.0` 的精确相似桶提高了 `8.2` 个百分点，solved rate 也由 `98.6%` 升至 `99.8%`。在 pairwise 层面，`full` 在 `135` 个问题上的 Tanimoto 高于 `compact`，反向仅有 `108` 个，另有 `257` 个持平。更准确的表述不是“full 更准”，而是更丰富的结构化上下文显著压低了劣质分解的比例，并提高了可用分解出现的概率。

同样关键的是，`full` 并没有让所有指标同步上升。它的严格 reference replay exact rate 反而从 `13.8%` 下降到 `11.6%`。这个细节不应被视为噪声，因为它恰好揭示了 richer context 的真实作用：它不是让模型更像 benchmark 一样复述参考前体，而是改变了模型在同一候选空间内部的选择方式。进一步看，增加的上下文使模型略微更保守，也更贴近已有工具层动作：`bond` 选择从 `80.6%` 上升到 `86.6%`，`fgi` 从 `18.6%` 下降到 `13.0%`，`custom` 从 `0.8%` 下降到 `0.4%`。从化学上看，这往往意味着模型更少被表面上也说得通的替代断法吸引，例如过早的 N-alkylation 或粗糙的局部重排，而更愿意保留与离去基、杂原子连接位点和局部取代环境一致的拆解。

![上下文暴露与选择质量的关系](/E:/Python/skills/Rachel/plan/figures/section6_fig2_context_vs_quality.svg)

*Fig. 2. `compact` 与 `full` 在同一候选空间下的上下文暴露与选择质量。A，输入侧的暴露差异。相对于 `compact`，`full` 不只是更长，而且显著增加了模型可见的结构化证据密度。B，输出侧的质量变化，横轴为 `full - compact`。`full` 提高了 mean/median Tanimoto、精确相似桶与 solved rate，并减少了低相似尾部；但严格的 reference replay exact 略有下降。这个组合结果支持的不是“上下文越多越像标准答案”，而是更丰富的结构化上下文改变了模型在既有候选空间中的选择方式。*

附录 Fig. A1 给出了一个足够典型的单步例子。对 `q0004` 而言，`compact` 只暴露了压缩后的 bond/FGI 摘要，模型因此落到一条 N-alkylation 路径上，Tanimoto 仅为 `0.279`；而在 `full` 条件下，完整结构化视图把 Williamson ether 断法清楚展开，同一个模型便恢复了真实前体，Tanimoto 达到 `1.0`。我们并不据此宣称所有样本都会出现如此整齐的改进，但这一案例足以说明 `full` 的主要机制：它并不改变化学基底，而是提高了模型在既有、已接地候选之间做出正确选择的概率。

因此，6.2 真正支持的不是一个泛泛的“LLM 有用”，而是一个更有边界感的判断：在 Rachel 中，LLM 的主要价值是策略性的，而不是生成性的。只有当局部决策被先行编译为结构化化学上下文，模型才真正表现出稳定收益；在这一框架里，工具层负责给出可检查的化学事实与候选动作，模型更像一个负责比较、排序和组织的策略层，而不是脱离约束独立给出答案的化学黑箱。

### 6.3 固定协议下，Rachel 路线表现出稳定闭合并伴随复杂度压缩

如果说前两节回答的是 Rachel 是否具备可用的局部动作空间，以及模型如何使用这一空间，那么端到端结果更直接地检验：这些局部动作能否在固定约束下累积为完整路线。本文的端到端分析聚焦 PaRoutes `n1` 的 `51` 个目标，其目的更接近于在受控条件下观察 Rachel 的路线行为，而不是对更大规模端到端性能下结论。

先看 benchmark 最标准的 solved-rate 口径，Rachel 并不是当前最强方法。Fig. 3 显示，PaRoutes、RootAligned、Rachel、RootAligned MCTS 和 LocalRetro 的 solved rate 分别为 `90.2%`、`86.3%`、`82.4%`、`80.4%` 和 `70.6%`。如果只按 solved fraction 排序，Rachel 的位置并不突出。真正值得进一步看的，是这 `82.4%` 背后对应的路线形态。与 solved rate 并列观察，Fig. 4 给出的 buyability profile 更能说明 Rachel 实际把路线推进到了哪里：在同一组 `51` 个目标上，`fully covered rate` 为 `82.4%`，`terminal pass rate` 为 `94.4%`，而 `trivial` 与 `moderate` terminal 的在线可买率分别为 `97.0%` 和 `79.4%`。这提示当前缺口主要不在于系统大面积停在上游拆解，而更多出现在较难终止物的接地。

更有信息量的是这些已解路线究竟长成什么样。为了避免把“解得更多”与“解得更深”混在一起，Fig. 5 只比较各方法共同解出的 `29` 个 `n1` 目标。在这一 shared solved subset 上，Rachel 的 top-route steps、reaction depth 和 unique terminal count 分别达到 `4.86`、`4.14` 和 `4.28`，都明显落在比较方法形成的 baseline span 之外。这说明 Rachel 的成功路线并不只是多走几步，而是更倾向于把目标继续拆到更深层，同时暴露出更丰富的终止物组合。就路线化学而言，这通常意味着系统没有停留在较早的中间体层面，而是继续把分子压向更接近可采购构件的状态。

*Fig. 5. shared solved `n1` 子集上的跨方法路线结构对比。图中仅统计被所有比较方法共同解出的 `29` 个目标，以避免 solved set 差异干扰结构比较。三个 panel 分别汇总 top-route steps、maximum reaction depth 和 unique terminal count。浅灰色带表示比较方法的数值区间，虚线表示其均值，绿色标出 Rachel。Rachel 在三个指标上都落在 baseline span 之外，说明其成功路线不仅更深，也终止于更丰富的 distinct terminal set。*

这一点在全部 `51` 个 `n1` 目标上也能看到。Rachel 为 `51/51` 个目标都生成了完整的原始路线树，平均 `5.35` 步，中位 `5` 步；平均最大深度 `4.51`，中位 `4`。至少在当前冻结设置下，路线展开能够稳定完成，而不是频繁停在局部建议阶段。换句话说，Rachel 输出的不是若干零散的单步分解，而是真正具有层级结构的多步路线。

更关键的是，这些路线并不只是“写出来”，而是相当一部分已经在当前审计口径下闭合。在线采购审计下，`42/51 = 82.35%` 条路线 fully covered；在终止分子层面，`231` 个 terminals 中有 `218` 个通过当前在线审计，对应 `94.37%` 的 terminal pass rate。与此同时，按 benchmark 导出规则被正式写入的 strict routes 数量同样是 `42`。因此，这里的闭合并不是事后放宽标准得到的乐观重计，而是在同一冻结规则下得到的结果。它当然不等同于真实实验室可执行性的充分证明，但至少说明 Rachel 已经能够在固定协议与预算下，把相当一部分目标推进到可接受终点。

如果把视角切回 reference-route replay，图景会明显不同。Exact tree、reaction set 与 terminal set 的 `Acc@1` 都只有 `1/51 = 1.96%`，已解路线上的 mean tree similarity 也只有 `25.59%`，median 为 `21.64%`。这些结果首先说明的不是 Rachel “做错了多少”，而是它走到终点的方式常常不同于 benchmark 中记下的那条路线。对合成规划而言，这一点并不意外：中间体选择、局部断裂顺序和终止物落点都可能变化，而最终路线仍然可以在当前口径下闭合。

因此，closure 与 replay 在这里对应的是两种不同信息。前者回答系统是否把目标分子持续推进到了可接受终点，后者回答它是否沿着 benchmark 所记录的那条路径前进。附录 Fig. A2 的 `n1_366` 给出了一个直观例子：参考路线为 `5` 步，Rachel 给出的路线为 `7` 步；两者在中间体和局部步骤组织上并不一致，但 Rachel 仍然落到了当前在线审计可接受的终止原料集合。这个个案不是为了宣称 Rachel 比参考路线更优，而是为了说明，一条路线如何被逐步展开、验证并最终闭合，与它是否回放了参考路径，在经验上确实是两个不同层面的判断。

Fig. 6 进一步说明，Rachel 的额外展开深度并不是空转。如果只看被多种方法共同解出的目标，它一方面具有更高的 mean terminal leaf depth，另一方面也伴随着更强的复杂度压缩：Bertz reduction ratio 达到 `81.0%`，CS reduction ratio 达到 `49.6%`，均高于图中比较方法大致 `66%–68%` 和 `39.6%–40.2%` 的区间；同时，其 terminal 的平均 heavy-atom count 与 ring count 也更低。化学上，这意味着 Rachel 的更深路线通常确实把目标压向了更小、更简单、环系更少的终止物，而不是仅仅把树做大。对本文而言，这比“路线更长”更重要，因为它说明额外深度换来的是真实的结构简化。

![common solved targets 上的复杂度压缩对比](/E:/Python/skills/MCTS-RETRO/benchmark/paroutes120/reports/complexity_analysis/figures/figureC1_common_solved_complexity_compression.png)

*Fig. 6. 在 common solved targets 上的跨方法复杂度压缩对比。左图显示 Rachel 在该公共可解子集上具有更高的 mean terminal leaf depth，同时保持更强的 Bertz complexity reduction；中图显示 Rachel 导向的 terminal 在 heavy-atom count 和 ring count 上更低；右图汇总了 CS ratio 与 Bertz ratio。该图说明，在多种系统都能解出的那部分目标上，Rachel 的更深路线同时伴随着更强的复杂度压缩，而不是单纯更长。*

### 6.4 当前未闭合主要集中在终止物接地，而非整条路线失效

如果说 6.3 说明 Rachel 已经能够把相当一部分目标推进到闭合终点，那么失败分析更关心剩下的样本卡在什么地方。对当前 `n1` 结果而言，最有解释力的不是先搭一个抽象的失败分类，而是直接看未闭合发生在路线的哪一层、集中在哪类 terminal，以及它更像上游规划问题还是下游接地问题。这样的分析也更符合本文当前的证据边界：先把失败位置看清，再对个别化学案例做更细分析。

| 失败视角 | 主要数值 | 暴露出的边界 |
| --- | ---: | --- |
| 路线闭合缺口 | `9/51` 条路线未 fully covered；其中 `6` 条只差 `1` 个 terminal，`2` 条差 `2` 个，`1` 条差 `3` 个 | 多数失败是 near-miss，而不是整条路线失效 |
| 缺失终止分子的原因 | 共 `13` 个缺失 terminal；其中 `9` 个为 `cid_unresolved`，`4` 个为 `no_vendor_listing` | 采购解析与供应映射仍是当前边界 |
| 类别分布 | `trivial`: `191/197 = 96.95%`；`moderate`: `27/34 = 79.41%`；`moderate` 仅占全部 terminals 的 `14.72%`，却占缺失项的 `53.85%` | 难点集中在更难采购的终止物上 |

表中的数字首先说明，多数未闭合路线已经非常接近闭合。`51` 条路线里只有 `9` 条未 fully covered，其中 `6` 条只差 `1` 个 terminal，另外 `2` 条差 `2` 个，只有 `1` 条差 `3` 个。就路线层级看，这更像 near-miss，而不是整条多步规划失效。也就是说，Rachel 在这些样本上通常已经把大部分路线写对了，只是最后的终止物还没有完全落稳。

缺口主要集中在终止物解析与供应映射。`13` 个缺失 terminal 中，`9` 个属于 `cid_unresolved`，`4` 个属于 `no_vendor_listing`。这提示当前未闭合样本里，相当一部分首先不是“路线在化学上走不通”，而是“路线已经走到候选终点，但该终点还没有在现有在线审计流程中稳定映射为可采购对象”。因此，就现阶段最直接的改进方向看，与其继续盲目扩搜索，不如优先把 terminal normalization、CID 解析和供应商映射做稳。

终止物难度分布进一步加强了这一判断。`trivial` 终止物的在线可采购率达到 `96.95%`，而 `moderate` 仅为 `79.41%`；更关键的是，`moderate` 虽只占全部终止物的 `14.72%`，却贡献了 `53.85%` 的缺失项。也就是说，失败并不是均匀地散落在整个终止物空间里，而是集中在一个更小、也更难的子集上。这种分布对后续改进是有利的，因为它说明问题已经被局部化，而不是平均扩散到所有终点类型。与此同时，Rachel 的粗分类与在线审计之间仍有 `198/231 = 85.71%` 的一致率，说明这套分类可以作为有效的先验筛分，但它不能替代真实采购审计。

因此，当前最值得保留的失败叙事并不是抽象地说系统“还有一些 unsolved”，而是更具体地指出：Rachel 已经能够稳定写出完整路线，当前最直接的短板主要落在终止物接地，尤其是较难采购的 `moderate` terminal；只有把这部分接地问题剥离出去之后，剩余样本中的局部规划偏置和算子覆盖缺口才值得被更细地逐案分析。对本文而言，这一点的重要性在于，失败不再只是一个总量上的剩余项，而开始能够被定位为路线状态中的具体停点。

### 6.5 本节小结

合起来看，第 6 节并不是几组松散实验的并列，而是一条围绕同一命题展开的证据链。全量单步结果说明，这套化学基底本身值得规划，而且 `bond` 与 `FGI` 作为两类结构探索算子能够共同扩大局部可达域；`compact` 与 `full` 的比较进一步表明，LLM 的主要作用不是替代化学工具自由生成前体，而是在已接地候选空间中改善选择与排序；最后，`n1` 上的端到端结果显示，Rachel 已经能够在固定协议下把相当一部分目标推进到实用意义上的闭合终点，而其当前边界更多暴露在终止物接地与较难 terminal 类别上。

因此，本节最核心的结论并不是某一个单一总指标，而是 Rachel 的几个关键设计在结果层面彼此呼应：先有可搜索的局部化学动作空间，再有建立在化学事实之上的策略选择，最后才是路线层面的闭合、偏离与失败定位。本文真正关心的，也不是“Rachel 是否足够像基准参考路线”，而是“它是否已经把多步逆合成写成了一种可以逐步展开、验证、分析并继续改进的路线推理过程”。

## 7. 讨论

综合以上结果：Rachel 的价值并不主要体现在把 solved rate 推到某个单一数字，而在于它为多步逆合成提供了一种更接近“形式化化学推理”的路线组织方式。这里所谓“形式化”，并不是要把合成规划写成逻辑证明，而是要把原本隐含的判断步骤拆开：目标如何被持续改写为路线状态，哪些局部动作值得尝试，哪些步骤通过了验证，哪些失败停在了终止物接地。就本文已有证据看，这种写法至少在三个层面显示出意义。

第一，它把多步逆合成从一次性的答案生成，提升为一个在化学约束下逐步构造、验证并提交路线的过程。对多步任务来说，这意味着路线不再只是终稿答案，而成为一个可以逐步分析的对象。

第二，现有结果支持一种较清楚的职责分配。`compact` 与 `full` 的比较表明，更丰富的结构化上下文确实提高了局部分解决策质量，但改进主要来自模型在已有 `bond / FGI` 候选中的选择，而不是来自大规模 `custom` 提议。就目前证据看，把化学工具放在事实和约束层、把 LLM 放在策略和排序层，是更贴近数据的解释。

第三，这样的写法也改变了多步逆合成中成功与失败的理解方式。在 `n1` 上，Rachel 虽然 strict replay 很低，但仍在当前在线审计下使 `42/51` 条路线达到闭合，并在共同可解目标上表现出更深的路线展开和更强的复杂度压缩。与此同时，未闭合样本又表明瓶颈更多落在终止物接地，而不是笼统的规划崩塌。这说明对多步任务而言，reference-route replay、practical closure 与 failure location 对应的是不同层面的信息。

因此，Rachel 更适合被看作一种带有明确局部动作和验证环节的路线表示，而不是一次性输出答案的黑箱生成器。本文真正想强调的，也不是它已经在所有评价口径上领先于现有方法，而是它让一条路线为何成立、又在何处中断，开始能够被更细致地讨论。

这一定性判断也必须放在当前系统边界内理解。Rachel 仍然继承了底层断键/FGI 底盘的覆盖限制，模板特异性与转化覆盖都还不完整；现有算子先验带有轻微的 bond-first 偏置，这在很多目标上有帮助，但在更适合 transformation-first 的案例中也可能错误分配预算；验证层仍是对化学可接受性的一种工程化近似，而不是完整的合成可行性模型；当前目标函数也主要是结构性的，尚未直接纳入成本、收率、鲁棒性或工艺尺度等实际因素。

同样需要说明的是，本文支持的结论，是 Rachel 作为一种带状态记录的多步路线表示，已经在局部动作覆盖、决策质量和冻结设置下的端到端闭合三个层面表现出稳定信号，而不是每一个系统组件的因果贡献都已被严格分离。组件级消融、更大规模的端到端目标集，以及结合具体失败实例的细粒度化学分析，仍然是后续工作的重点。更准确的说法不是 Rachel 已经解决了多步逆合成，而是它提供了一种值得继续扩展和系统检验的路线表示。

## 8. 结论

本文提出 Rachel，将多步逆合成组织为一个 `state -> action -> validation -> commit` 循环，而不再把它简化为无约束的路线生成任务。当前结果支持三个相对克制但清楚的结论：由 `bond + FGI` 构成的化学底盘在单步语料上提供了稳定的局部覆盖；更丰富的化学上下文主要改善了 LLM 的选择质量，而不是让模型脱离工具约束去提出新化学；在端到端多步场景中，Rachel 已经能够在固定设置下闭合相当一部分目标，同时把未闭合轨迹保留为可继续分析的路线记录。

因此，Rachel 的主要贡献不在于把多步逆合成重新包装成某一个单一总分上的竞争，而在于给出了一种更清楚的路线表示：化学工具负责事实与可接受性，LLM 负责在这些信息之上组织策略，失败也不再只是被隐藏的噪声，而成为后续分析和系统改造的线索。就本文所覆盖的证据范围而言，更准确的定位不是 Rachel 已经完备地解决了多步逆合成，而是它为多步规划提供了一种可以继续扩展、也可以继续检验的路线表示。
